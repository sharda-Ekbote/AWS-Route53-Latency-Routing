
AWS-Route53-Latency-Routing

 To configure Latency-Based Routing using Amazon Route 53. Latency routing allows you to improve your application's performance by
 routing user traffic to the AWS region that provides the lowest network latency (the fastest response time).

Architecture Components
**Amazon Route 53: The highly available Domain Name System (DNS) web service.

**Hosted Zone: A container for DNS records for a specific domain.

**Latency Routing Policy: A mechanism that directs traffic based on the geographic latency between the user and AWS regions.
